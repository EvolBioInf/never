      \section{docs}
      \ty{docs} is the frontend application for never api/v2.
      It serves to document all endpoints, requests, responses and
      schemes provided by the api. Data is provided by a openAPI
      document in json format. While based on go's html templating
      package it uses basic html, css and javscript to render the docs.

      \subsection*{Implementation}
      The program \ty{docs} provides functions for registering the routes
      and obtaining the api data from the openAPI document, as
      well as types for holding the data in intermediate data structures.

#+begin_src latex
  Now we define \ty{toCSSid}. The function is provided a variable
  amount of strings and returns a single joined string. To get
  rid of the hierarchical url structure of an endpoint we replace
  every slash with a dash whichr results in strings in kebab-case notation.
  Then we join the strings and delete leading dashes as well as dashes followed
  by another dash.
#+end_src

#+begin_src go <<Functions, Pa. \ref{pa:docs}>>
  func toCSSId(args ...string) string {
  	noSlash := make([]string, len(args))
  	for i, str := range args {
  		noSlash[i] = strings.ReplaceAll(str, "/", "-")
  	}
  
  	joined := strings.ReplaceAll(strings.Join(noSlash, "-"), "--", "-")
  	if len(joined) > 1 && joined[0] == '-' {
  		joined = joined[1:]
  	}
  
  	return joined
  }

#+begin_src latex
  The next step is to retrieve the data from the openAPI document.
  For this we use the library libopenapi. This library parses a
  byte slice from an openAPI document and lets us create a \ty{DocumentModel}.
  Using this model we can retrieve the data and form it into an intermediate
  representation. The intermediate representation needs to be taylored
  to our spec and use for it within the html templating. With this in mind
  we create the types that shall hold our data.
